css负边距与圣杯布局（双飞翼布局）

发表于 2017-10-08 | 分类于 css |

大家常说的圣杯布局也就是双飞翼布局，指的是两侧固定宽度，中间自适应的三栏布局，淘宝首页采用的就是这种圣杯布局。

此外，这种布局还有个特点就是，无论随着页面宽度怎么变化，中间的自适应区域是优先渲染的，两边的盒子框子固定不变，即使页面宽度变小，也不影响我们的浏览。

根据样式我们可以看到，三个盒子都包含在同一个container 容器中，并且该容器设置了overflow: hidden，形成了一个BFC，将三个设置了左浮动的盒子包含起来（参见BFC 的清除浮动特性），如果不触发容器的BFC ，那么就会有尴尬的浮动引起父元素塌陷问题。三个盒子都设置了左浮动，按理说应该是在同一行的，但是因为我们又把中间的盒子的宽度设为了100%，并且它是第一个渲染的元素，所以硬是把左右栏盒子给挤下去了。

要让左右栏盒子与中间盒子并排，就需要用到神奇的负边距了。左栏盒子需要回到最左侧，所以给左栏盒子添加margin-left: -100%，它就会回到上一行了。同理，因为右边栏盒子要回到的是上一栏的右侧，所以只需要将其margin-left 设为自己的负宽度就行margin-left: -200px。

此时的效果：

现在三栏终于在同一行了，看上去好像大功告成了，但是，仔细一看可以会发现中间盒子的内容其实被遮挡住了，不信的话可以往中间盒子中填充更多的内容，就会发现这个问题。

为了让中间自适应的盒子可以安全显示，我们还要进行下面的操作。首先，给父级元素，也就是前面提到过的触发了BFC 的container 设置左右内边距.container{ padding: 0 200px}，设多少呢？设200px，刚好就是左右盒子他们各自的宽度。

这时可以看到因为给容器设了左右padding的原因，三个盒子往中间挤了，但是中间盒子还是被左右栏盖住，别急，继续操作。

我们给左右栏盒子增加相对定位的样式：.left {position: relative; left: -200px}和.right{position: relative; right: -200px}

OK，布局就这么完成了。

用flex 做圣杯布局

当然我们也可以用更现代的flex 方法去做圣杯布局。首先将header，中间以及footer 放置在一个容器中，该容器设置成垂直方向的flex 布局，另外还要给header 和footer 设置固定高度，中间则设置为自动高度（即由其子内容撑开决定）。

然后中间部分我们也把他设置成flex 容器，左中右三栏放置在其中，设置成横向flex 布局。左栏和右栏设置固定宽度，并且通过修改order 属性把左栏成为本列首位的元素，中间部分的设置为自适应。这就OK了，只要熟悉原理，flex 方法实现看起来的确要比传统方法优雅很多啊。
